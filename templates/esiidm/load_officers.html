Officer bulk load

An officer is just a person that has special status and is linked to a HEI (or more) whose cards and students is authorised to manage.
The first officer for a HEI may be loaded as an unlinked person and the link is later established when the HEI is created.

The CSV file MUST (RFC2119) have field names as columns on the first row.
Fields MAY (RFC2119) be in any order.

The CSV file MUST (RFC2119) have a minimum set of fields:

  • first_name: officer's given name
  • last_name: officer's family name
  • email: officer's email

The rows MAY (RFC2119) also contain these optional fields:

  • identifier: government issued identifier for the person
  • pic: officer's HEI PIC code
  • euc: officer's HEI EUC code
  • erc: officer's HEI ERC code

Only one of PIC, EUC or ERC is required for linking the officer to the HEI.
If there is no HEI, the person is created in the system with needed privileges but no associated HEI.
Persons receive an invitation via email for activating the account.
